A small and simple integration.\hypertarget{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_client_node_modules_copy_anything__r_e_a_d_m_e_autotoc_md3535}{}\doxysection{Motivation}\label{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_client_node_modules_copy_anything__r_e_a_d_m_e_autotoc_md3535}
I created this package because I tried a lot of similar packages that do copy\textquotesingle{}ing/cloning. But all had its quirks, and {\itshape all of them break things they are not supposed to break}... 😞

I was looking for\+:


\begin{DoxyItemize}
\item a simple copy/clone function
\item has to be fast!
\item props must lose any reference to original object
\item works with arrays and objects in arrays!
\item supports symbols
\item can copy non-\/enumerable props as well
\item $\ast$$\ast$does not break special class instances$\ast$$\ast$　‼️
\end{DoxyItemize}

This last one is crucial! So many libraries use custom classes that create objects with special prototypes, and such objects all break when trying to copy them inproperly. So we gotta be careful!

copy-\/anything will copy objects and nested properties, but only as long as they\textquotesingle{}re \char`\"{}plain objects\char`\"{}. As soon as a sub-\/prop is not a \char`\"{}plain object\char`\"{} and has a special prototype, it will copy that instance over \char`\"{}as is\char`\"{}. ♻️\hypertarget{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_client_node_modules_copy_anything__r_e_a_d_m_e_autotoc_md3536}{}\doxysection{Meet the family}\label{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_client_node_modules_copy_anything__r_e_a_d_m_e_autotoc_md3536}
