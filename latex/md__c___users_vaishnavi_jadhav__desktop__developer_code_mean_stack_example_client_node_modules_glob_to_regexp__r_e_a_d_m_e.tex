Turn a $\ast$-\/wildcard style glob ({\ttfamily \char`\"{}$\ast$.\+min.\+js\char`\"{}}) into a regular expression ({\ttfamily /$^\wedge$.$\ast$\textbackslash{}.min\textbackslash{}.js\$/})!

To match bash-\/like globs, eg. {\ttfamily ?} for any single-\/character match, {\ttfamily \mbox{[}a-\/z\mbox{]}} for character ranges, and {\ttfamily \{$\ast$.html, $\ast$.js\}} for multiple alternatives, call with {\ttfamily \{ extended\+: true \}}.

To obey \href{https://github.com/isaacs/node-glob\#glob-primer}{\texttt{ globstars {\ttfamily $\ast$$\ast$}}} rules set option {\ttfamily \{globstar\+: true\}}. NOTE\+: This changes the behavior of {\ttfamily $\ast$} when {\ttfamily globstar} is {\ttfamily true} as shown below\+: When {\ttfamily \{globstar\+: true\}}\+: {\ttfamily /foo/$\ast$$\ast$} will match any string that starts with {\ttfamily /foo/} like {\ttfamily /foo/index.htm}, {\ttfamily /foo/bar/baz.txt}, etc. Also, {\ttfamily /foo/$\ast$$\ast$/$\ast$.txt} will match any string that starts with {\ttfamily /foo/} and ends with {\ttfamily .txt} like {\ttfamily /foo/bar.txt}, {\ttfamily /foo/bar/baz.txt}, etc. Whereas {\ttfamily /foo/$\ast$} (single {\ttfamily $\ast$}, not a globstar) will match strings that start with {\ttfamily /foo/} like {\ttfamily /foo/index.htm}, {\ttfamily /foo/baz.txt} but will not match strings that contain a {\ttfamily /} to the right like {\ttfamily /foo/bar/baz.txt}, {\ttfamily /foo/bar/baz/qux.dat}, etc.
