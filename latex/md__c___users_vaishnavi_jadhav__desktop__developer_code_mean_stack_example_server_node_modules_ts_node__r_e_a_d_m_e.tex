ts-\/node is a Type\+Script execution engine and REPL for Node.\+js.

It JIT transforms Type\+Script into Java\+Script, enabling you to directly execute Type\+Script on Node.\+js without precompiling. This is accomplished by hooking node\textquotesingle{}s module loading APIs, enabling it to be used seamlessly alongside other Node.\+js tools and libraries.\hypertarget{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_server_node_modules_ts_node__r_e_a_d_m_e_autotoc_md18950}{}\doxysubsection{Features}\label{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_server_node_modules_ts_node__r_e_a_d_m_e_autotoc_md18950}

\begin{DoxyItemize}
\item Automatic sourcemaps in stack traces
\item Automatic {\ttfamily tsconfig.\+json} parsing
\item Automatic defaults to match your node version
\item Typechecking (optional)
\item REPL
\item Write standalone scripts
\item Native ESM loader
\item Use third-\/party transpilers
\item Use custom transformers
\item Integrate with test runners, debuggers, and CLI tools
\item Compatible with pre-\/compilation for production
\end{DoxyItemize}

\hypertarget{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_server_node_modules_ts_node__r_e_a_d_m_e_autotoc_md18959}{}\doxysubsection{Via tsconfig.\+json (recommended)}\label{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_server_node_modules_ts_node__r_e_a_d_m_e_autotoc_md18959}
ts-\/node automatically finds and loads {\ttfamily tsconfig.\+json}. Most ts-\/node options can be specified in a {\ttfamily \char`\"{}ts-\/node\char`\"{}} object using their programmatic, camel\+Case names. We recommend this because it works even when you cannot pass CLI flags, such as {\ttfamily node -\/-\/require ts-\/node/register} and when using shebangs.

Use {\ttfamily -\/-\/skip\+Project} to skip loading the {\ttfamily tsconfig.\+json}. Use {\ttfamily -\/-\/project} to explicitly specify the path to a {\ttfamily tsconfig.\+json}.

When searching, it is resolved using \href{https://www.typescriptlang.org/docs/handbook/tsconfig-json.html}{\texttt{ the same search behavior as {\ttfamily tsc}}}. By default, this search is performed relative to the entrypoint script. In {\ttfamily -\/-\/cwd\+Mode} or if no entrypoint is specified -- for example when using the REPL -- the search is performed relative to {\ttfamily -\/-\/cwd} / {\ttfamily process.\+cwd()}.

If no {\ttfamily tsconfig.\+json} is loaded from disk, ts-\/node will use the newest recommended defaults from \href{https://github.com/tsconfig/bases/}{\texttt{ @tsconfig/bases}} compatible with your {\ttfamily node} and {\ttfamily typescript} versions. With the latest {\ttfamily node} and {\ttfamily typescript}, this is \href{https://github.com/tsconfig/bases/blob/master/bases/node16.json}{\texttt{ {\ttfamily @tsconfig/node16}}}.

Older versions of {\ttfamily typescript} are incompatible with {\ttfamily @tsconfig/node16}. In those cases we will use an older default configuration.
