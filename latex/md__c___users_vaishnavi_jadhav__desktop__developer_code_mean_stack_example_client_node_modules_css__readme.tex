Accepts a CSS string and returns an AST {\ttfamily object}.

{\ttfamily options}\+:


\begin{DoxyItemize}
\item silent\+: silently fail on parse errors.
\item source\+: the path to the file containing {\ttfamily css}. Makes errors and source maps more helpful, by letting them know where code comes from.
\end{DoxyItemize}\hypertarget{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_client_node_modules_css__readme_autotoc_md4191}{}\doxysubsection{css.\+stringify(object, \mbox{[}options\mbox{]})}\label{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_client_node_modules_css__readme_autotoc_md4191}
Accepts an AST {\ttfamily object} (as {\ttfamily css.\+parse} produces) and returns a CSS string.

{\ttfamily options}\+:


\begin{DoxyItemize}
\item indent\+: the string used to indent the output. Defaults to two spaces.
\item compress\+: omit comments and extraneous whitespace.
\item sourcemap\+: return a sourcemap along with the CSS output. Using the {\ttfamily source} option of {\ttfamily css.\+parse} is strongly recommended when creating a source map. Specify `sourcemap\+: \textquotesingle{}generator'\`{} to return the Source\+Map\+Generator object instead of serializing the source map.
\item input\+Sourcemaps\+: (enabled by default, specify {\ttfamily false} to disable) reads any source maps referenced by the input files when generating the output source map. When enabled, file system access may be required for reading the referenced source maps.
\end{DoxyItemize}\hypertarget{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_client_node_modules_css__readme_autotoc_md4192}{}\doxysubsection{Example}\label{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_client_node_modules_css__readme_autotoc_md4192}
