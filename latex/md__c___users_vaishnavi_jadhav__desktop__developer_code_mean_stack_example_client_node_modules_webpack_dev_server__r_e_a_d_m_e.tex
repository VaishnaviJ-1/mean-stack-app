While it\textquotesingle{}s recommended to run webpack-\/dev-\/server via the CLI, you may also choose to start a server via the API.

See the related \href{https://webpack.js.org/api/webpack-dev-server/}{\texttt{ API documentation for {\ttfamily webpack-\/dev-\/server}}}.\hypertarget{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_client_node_modules_webpack_dev_server__r_e_a_d_m_e_autotoc_md15427}{}\doxysubsubsection{The Result}\label{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_client_node_modules_webpack_dev_server__r_e_a_d_m_e_autotoc_md15427}
Either method will start a server instance and begin listening for connections from {\ttfamily localhost} on port {\ttfamily 8080}.

webpack-\/dev-\/server is configured by default to support live-\/reload of files as you edit your assets while the server is running.

See \href{https://webpack.js.org/configuration/dev-server/\#devserver}{\texttt{ {\bfseries{the documentation}}}} for more use cases and options.\hypertarget{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_client_node_modules_webpack_dev_server__r_e_a_d_m_e_autotoc_md15428}{}\doxysubsection{Browser Support}\label{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_client_node_modules_webpack_dev_server__r_e_a_d_m_e_autotoc_md15428}
While {\ttfamily webpack-\/dev-\/server} transpiles the client (browser) scripts to an ES5 state, the project only officially supports the {\itshape last two versions of major browsers}. We simply don\textquotesingle{}t have the resources to support every whacky browser out there.

If you find a bug with an obscure / old browser, we would actively welcome a Pull Request to resolve the bug.\hypertarget{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_client_node_modules_webpack_dev_server__r_e_a_d_m_e_autotoc_md15429}{}\doxysubsection{Support}\label{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_client_node_modules_webpack_dev_server__r_e_a_d_m_e_autotoc_md15429}
