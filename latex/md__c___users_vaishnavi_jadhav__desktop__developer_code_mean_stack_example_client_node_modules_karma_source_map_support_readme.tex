Karma plugin for inline sourcemap support.\hypertarget{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_client_node_modules_karma_source_map_support_readme_autotoc_md7619}{}\doxysection{Motivation}\label{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_client_node_modules_karma_source_map_support_readme_autotoc_md7619}
When loading Browserify bundles with inline source maps (via \href{https://www.npmjs.com/package/karma-browserify}{\texttt{ {\ttfamily karma-\/browserify}}}), the stack traces in Chrome don\textquotesingle{}t mention the original modules. This plugin uses \href{https://www.npmjs.com/package/source-map-support}{\texttt{ {\ttfamily source-\/map-\/support}}} to improve the situation.\hypertarget{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_client_node_modules_karma_source_map_support_readme_autotoc_md7620}{}\doxysection{Use}\label{md__c___users_vaishnavi_jadhav__desktop__developer_code_mean_stack_example_client_node_modules_karma_source_map_support_readme_autotoc_md7620}
Install the plugin with {\ttfamily npm}\+: \begin{DoxyVerb}npm install karma-source-map-support
\end{DoxyVerb}
 Configure Karma to load the plugin as a framework\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{module.exports = function(config) \{}
\DoxyCodeLine{  config.set(\{}

\DoxyCodeLine{    // additional settings here ...}

\DoxyCodeLine{\};}

\end{DoxyCode}
