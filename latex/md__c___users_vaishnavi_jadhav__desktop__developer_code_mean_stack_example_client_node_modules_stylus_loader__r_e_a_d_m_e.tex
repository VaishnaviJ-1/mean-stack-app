Using {\ttfamily $\sim$} is deprecated and can be removed from your code ({\bfseries{we recommend it}}), but we still support it for historical reasons. Why you can removed it? The loader will first try to resolve {\ttfamily @import}/{\ttfamily @require} as relative, if it cannot be resolved, the loader will try to resolve {\ttfamily @import}/{\ttfamily @require} inside \href{https://webpack.js.org/configuration/resolve/\#resolve-modules}{\texttt{ {\ttfamily node\+\_\+modules}}}.
